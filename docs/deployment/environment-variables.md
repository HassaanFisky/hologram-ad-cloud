# Environment Variables Reference

This document describes every environment variable used by the HoloLED Cloud platform, organized by service.

> **Rule:** Never commit real secrets to the repository. Use `.env.example` as the template for local development and a secrets manager (AWS Secrets Manager, HashiCorp Vault, or equivalent) for production.

---

## API (`apps/api`)

| Variable | Required | Default | Description |
|---|---|---|---|
| `NODE_ENV` | No | `development` | Runtime environment: `development`, `test`, or `production` |
| `DATABASE_URL` | **Yes** | ‚Äî | PostgreSQL connection string |
| `REDIS_URL` | **Yes** | ‚Äî | Redis connection string for queuing and cache |
| `JWT_ACCESS_SECRET` | **Yes** | ‚Äî | HMAC secret for signing access tokens (min 32 chars) |
| `JWT_REFRESH_SECRET` | **Yes** | ‚Äî | HMAC secret for signing refresh tokens (min 32 chars) |
| `S3_ENDPOINT` | **Yes** | ‚Äî | Object storage endpoint URL |
| `S3_REGION` | **Yes** | ‚Äî | Object storage region |
| `S3_BUCKET` | **Yes** | ‚Äî | S3 bucket name for media and OTA artifacts |
| `S3_ACCESS_KEY` | **Yes** | ‚Äî | S3 access key |
| `S3_SECRET_KEY` | **Yes** | ‚Äî | S3 secret key |
| `S3_FORCE_PATH_STYLE` | No | `true` | Use path-style URLs (required for MinIO) |
| `MQTT_URL` | **Yes** | ‚Äî | MQTT broker URL (e.g., `mqtt://emqx:1883`) |
| `MQTT_USERNAME` | No | ‚Äî | MQTT broker username |
| `MQTT_PASSWORD` | No | ‚Äî | MQTT broker password |
| `API_PORT` | No | `4000` | Port the API listens on |
| `WEB_ORIGIN` | **Yes** | ‚Äî | Allowed CORS origin for the operator dashboard |

## Production Security Rules

- Access tokens must use a secret of at least 64 characters generated by a CSPRNG
- Refresh token secret must differ from access token secret
- Database credentials must be rotated after every engineer departure
- S3 credentials must use IAM roles with least-privilege bucket policies in production
- MQTT device credentials must be per-device and revocable
- All secrets must be stored in a secrets manager, not in files or CI environment variables that log to stdout
